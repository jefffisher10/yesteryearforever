# yesteryearforever.xyz

A minimalist personal website.

This site is a small collection of writing, projects, observations, and experiments.
It is intentionally simple: plain HTML pages, a single CSS file, and a few small Python scripts that automate repetitive tasks.

The goal is for the site to remain easy to understand, maintain, and edit by hand.

---

# Philosophy

This site avoids complex frameworks or static site generators.

Instead it uses:

* plain HTML pages
* one stylesheet
* one JavaScript file
* a small Python utility script

The result is a website that can still function even if the automation scripts are never run again.

Everything is transparent and easy to repair manually.

# Important Files

**index.html**
Main landing page with site description and category showcase cards.

**ABOUT.html**
Hybrid résumé and autobiography. Uses its own scoped styles inside the file.

**template.html**
Template for new posts. Includes placeholder comments for all common
image layouts and the required category/desc meta comments.

**writing.html / making.html / sensing.html**
Category index pages. These are automatically regenerated by `update.py`
each time it runs. Do not edit them by hand — changes will be overwritten.

**style.css**
Main styling for the entire site including the top nav, category list
dot-leaders, dark mode, lightbox, gallery grid, and mobile layout.

**script.js**
Handles UI features:

* random post button (pages list auto-updated by update.py)
* dark mode toggle with localStorage persistence
* image lightbox
* audio playlists

**update.py**
Master utility script. Run this after adding any new post or image.

**inject_meta.py**
One-time migration script used to add category/desc comments to existing
posts. Can be re-run safely — skips already-tagged files.

# Creating a New Post

1. Copy `template.html` and rename it.

Example:
```
my_new_post.html
```

2. Set the page title:
```html
<title>My Post Title</title>
```

3. Set the category and description near the top of `<body>`:
```html
<!-- category: writing -->
<!-- desc: one short line describing this post -->
```

Valid categories: `writing` · `making` · `sensing`

4. Write the content inside `<blockquote>`.

**Important HTML rule:** never put images inside a `<p>` tag.
Each paragraph is its own `<p>`. Images sit between paragraphs as siblings.

5. Add the date at the bottom:
```html
<p style="font-size: 0.85rem; color: #999;">YYYY-MM-DD</p>
```

6. Add images to the `images/` directory.

7. Run `update.py`.

# Image Layouts

All image patterns are shown in `template.html`. The three main layouts:

**Float right or left:**
```html
<figure>
  <img src="images/FILENAME_thumb.jpg"
       data-full="images/FILENAME.jpg"
       alt="description"
       class="thumbnail align-right"
       loading="lazy">
  <figcaption>Optional caption.</figcaption>
</figure>
```

**Side by side:**
```html
<div class="side-by-side">
  <img src="images/A_thumb.jpg" class="thumbnail" data-full="images/A.jpg" loading="lazy">
  <img src="images/B_thumb.jpg" class="thumbnail" data-full="images/B.jpg" loading="lazy">
</div>
```

**Gallery grid:**
```html
<div class="gallery">
  <img src="images/A_thumb.jpg" data-full="images/A.jpg" class="thumbnail" loading="lazy">
  <img src="images/B_thumb.jpg" data-full="images/B.jpg" class="thumbnail" loading="lazy">
  ...
</div>
```

Thumbnails are generated automatically by `update.py`. You can use the
full filename in the HTML — the script will add the `_thumb` version.

# Running the Update Script

After adding new posts or images:

```
python3 update.py
```

This script performs the following tasks in order:

1. Generates image thumbnails (`_thumb` versions)
2. Updates the nav bar across all post pages (seeing → sensing, etc.)
3. Updates `<img>` tags to use thumbnail src with full-size data-full
4. Updates the random post list in `script.js`
5. Rebuilds `writing.html`, `making.html`, and `sensing.html` from post meta comments
6. Generates `rss.xml`
7. Generates `sitemap.xml`
8. Fixes file permissions

# Category System

Each post file contains two HTML comments that the update script reads:

```html
<!-- category: writing -->
<!-- desc: one short line describing this post -->
```

These drive the automatic category page generation. The script:

* scans all post files for these comments
* groups posts by category
* sorts by date (newest first; undated posts sink to bottom)
* rebuilds the category index pages

To add a post to a category, just add the comments and run `update.py`.

# Navigation

The top nav bar appears on every page:

```
| about | making | writing | sensing | dark mode | random |
```

The nav is defined in `NAV_HTML` inside `update.py` and is injected/updated
across all post pages automatically when the script runs. To change the nav
globally, update `NAV_HTML` in `update.py` and run the script.
